🎓
학생 (Student)
클래스 코드로 수업에 입장하며, 자신의 워크북만 작성·조회할 수 있습니다.
  • 클래스 코드로 수업 입장
  • 본인 워크북 작성 및 저장
  • 본인 답변 최종 제출
  • 수업 진행 현황 조회
👩‍🏫
교사 (Teacher)
자신이 담당하는 반의 수업을 진행하고 학생 응답을 실시간 모니터링합니다.
  • 수업 세션 생성 및 제어
  • 담당 학생 전체 응답 조회
  • 문항 활성화·비활성화
  • 전자칠판 화면 제어
  • 응답 공개·비공개 설정
🏫
관리자 (Admin)
학교 단위 계정을 발급하고 전체 플랫폼 사용 현황을 관리합니다.
  • 학교 계정 발급 및 삭제
  • 교사·학생 계정 일괄 관리
  • 전체 사용 통계 조회
  • 플랜 관리 및 설정 변경
RLS (Row Level Security) 정책 예시Supabase PostgreSQL
-- 학생 응답: 본인 데이터만 읽기/쓰기 CREATE POLICY "students_own_responses" ON workbook_responses FOR ALL USING ( student_id = auth.uid() ); -- 교사: 담당 클래스 학생 응답만 읽기 CREATE POLICY "teachers_class_responses" ON workbook_responses FOR SELECT USING ( class_id IN ( SELECT id FROM classes WHERE teacher_id = auth.uid() ) ); -- 관리자: 소속 학교 전체 데이터 접근 CREATE POLICY "admin_school_access" ON workbook_responses FOR ALL USING ( (auth.jwt() ->> 'role') = 'admin' AND school_id = (auth.jwt() ->> 'school_id')::uuid );
