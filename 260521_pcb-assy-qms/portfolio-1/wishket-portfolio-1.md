## 프로젝트 상세 설명 (description, 최대 5000자)

1) 포트폴리오 소개
PCB A'ssy 제조업 현장의 입고·공정·출하 검사 전 과정을 디지털화하는 품질관리시스템(QMS)을 React PWA + Node.js 기반으로 설계·구현합니다. AQL 샘플링 자동 산출, 불량 내역 실시간 입력, 결재 워크플로우, PDF 성적서 자동 생성까지 제조업 프로세스 디지털화를 위한 MVP 데모를 제공합니다.

2) 작업 범위
- 실시간 불량 입력 및 검사 현황 대시보드: 태블릿에서 불량 내역을 실시간으로 입력하고, 라인별·공정별 불량률을 한눈에 파악할 수 있는 통합 대시보드를 제공합니다.
- AQL 샘플링 자동 산출 및 협력사 기준 관리: 협력사별 검사 기준을 설정하고, 입고 수량에 따라 AQL 테이블(MIL-STD-1916 기준)을 자동으로 참조하여 샘플 수와 합격/불합격 기준을 즉시 산출합니다.
- 결재 워크플로우 및 조건부 합격 승인: 검사 결과에 따라 합격·조건부 합격·불합격 판정을 구분하고, 조건부 합격 건에 대해 품질관리자 → 팀장 단계별 결재 워크플로우를 구현합니다.
- PDF 성적서 자동 생성 및 이메일 발송: 승인 완료된 검사 건에 대해 회사 양식에 맞는 검사 성적서 PDF를 자동 생성하고, 협력사 담당자에게 이메일로 자동 발송하는 기능을 제공합니다.

3) 주요 업무
- 실시간 불량 입력 및 검사 현황 대시보드 화면: React PWA 기반 오프라인 지원 입력 폼, WebSocket 실시간 집계, Recharts 기반 불량률 차트 구현으로 현장 검사자와 관리자가 동일한 데이터를 실시간으로 공유합니다.
- AQL 샘플링 자동 산출 및 협력사 기준 관리 화면: AQL 레벨별 테이블을 DB에 구조화하여 입고 수량 입력만으로 샘플 수가 자동 계산되고, 협력사별 AQL 등급 설정을 관리자 화면에서 유연하게 관리할 수 있도록 설계합니다.
- 결재 워크플로우 및 조건부 합격 승인 화면: 단계별 결재 상태를 DB에 FSM(유한 상태 기계) 패턴으로 관리하고, 결재 대기/완료 건을 담당자별 인박스로 제공하며, 이메일/Slack 알림 연동으로 결재 지연을 방지합니다.
- PDF 성적서 자동 생성 및 이메일 발송 화면: Puppeteer 기반 HTML→PDF 변환으로 픽셀 수준 레이아웃을 유지하고, 성적서 번호 채번, AWS S3 저장, Nodemailer 이메일 발송까지 단일 API 호출로 처리합니다.

4) 주안점
- React Query의 optimistic update 패턴과 WebSocket 이벤트를 결합하여 태블릿 입력 즉시 대시보드에 반영되는 실시간 동기화 구조를 설계합니다.
- 협력사별 AQL 기준을 동적으로 참조하는 계산 엔진을 순수 함수로 설계하여, 기준 변경 시 과거 검사 이력은 보존하면서 신규 검사에만 새 기준이 적용되는 버전 관리 구조를 구현합니다.
- 결재 상태 전이를 서버 단의 트랜잭션과 낙관적 잠금으로 보호하여 동시 결재 충돌이 발생하지 않는 안전한 워크플로우 엔진을 설계하고, 결재 이력 전체를 변경 불가 append-only 로그로 저장합니다.

※ 본 포트폴리오는 해당 도메인의 핵심 기능을 직접 설계·구현한 구현 제안 데모입니다. 기획부터 DB 설계, API, 프론트엔드까지 전 과정을 1인 풀스택으로 구현하며, 결과물 URL에서 실제 동작을 확인하실 수 있습니다. 프로젝트 착수 시 이 데모를 기반으로 빠르게 개발을 진행할 수 있습니다.

### 프로젝트 배경 (project_background, 최대 5000자)

1) 예상 리스크 및 대응 계획
- 협력사마다 다른 AQL 등급과 검사 기준을 버전 관리하여 기존 이력 데이터와 신규 기준 산출이 충돌하지 않는 이력 보존 구조 설계 필요
- 태블릿 네트워크 불안정 환경에서 오프라인 불량 입력 데이터가 유실 없이 서버와 동기화되는 충돌 해결 전략 설계 필요

- 한국/베트남 현장의 복합 문자(한글·베트남어·특수기호) 환경에서 PDF 성적서 렌더링 시 폰트 깨짐 없이 처리하는 Puppeteer 설정 필요
- 수만 건의 검사 이력을 효율적으로 조회하고 파레토 분석 집계를 빠르게 처리하는 PostgreSQL 인덱싱·Materialized View 설계 필요

### 프로젝트 성과 (achievements)

#### 성과 1
- 성과명: AQL 샘플링 자동 산출 엔진 설계
- 설명: MIL-STD-1916 기반 AQL 테이블 PostgreSQL 구조화, 협력사별 기준 버전 관리 구조로 과거 이력 보존과 신규 기준 분리 적용을 동시에 구현합니다.

#### 성과 2
- 성과명: 결재 FSM 트랜잭션 워크플로우 구축
- 설명: PostgreSQL Advisory Lock 기반 동시 결재 충돌 방지, 4단계 RBAC 권한 계층 설계, 결재 이력 append-only 로그로 위변조 불가 감사 추적을 구현합니다.

#### 성과 3
- 성과명: Puppeteer PDF 성적서 파이프라인 구현
- 설명: HTML 템플릿→PDF 변환 시 한글 폰트 임베딩·페이지 분할 처리, Bull Queue 재시도 정책(3회, exponential backoff)으로 미발송 방지 구조를 설계합니다.

#### 성과 4
- 성과명: PWA 오프라인 불량 입력 동기화 구현
- 설명: Service Worker + IndexedDB 기반 오프라인 입력 큐를 설계, 네트워크 복구 시 충돌 없는 단방향 머지 전략으로 현장 데이터 유실 방지를 구현합니다.
