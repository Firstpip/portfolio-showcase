## 프로젝트 상세 설명 (description, 최대 5000자)

1) 포트폴리오 소개
PCB A'ssy 제조업의 한국·베트남 다중 법인 데이터 통합 집계와 품질관리시스템(QMS) 확장 플랫폼을 Node.js + Redis + PostgreSQL 기반으로 설계·구현합니다. 실시간 불량률 모니터링, 파레토 분석, Slack 알림, Google 도구 연동, QR 코드 기반 표준문서 열람까지 제조업 프로세스 디지털화의 전체 로드맵을 데모로 제시합니다.

2) 작업 범위
- 다중 법인 통합 모니터링 대시보드: 한국·베트남 법인의 불량률·AQL 합격률·성적서 발행 현황을 단일 화면에 통합하여 경영진과 품질 책임자가 글로벌 품질 지표를 실시간으로 파악할 수 있습니다.
- 파레토 분석 및 임계치 알림 시스템: 불량 유형별 발생 빈도를 파레토 차트로 분석하고, 설정된 임계치를 초과하는 불량률 또는 반복 불량 유형을 감지하여 담당자에게 Slack과 이메일로 즉시 알림을 발송합니다.
- Google Sheets 연동 및 데이터 수집 자동화: 기존 Google Form·Sheets 기반 품질 데이터를 QMS 시스템으로 자동 수집·동기화하여 현장 전환 과도기에 이중 입력 없이 두 시스템이 공존할 수 있는 연동 구조를 제공합니다.
- QR 코드 기반 표준문서 열람 포털: 현장에 부착된 QR 코드를 스캔하면 해당 공정의 작업 표준서·검사 기준서를 태블릿·스마트폰에서 즉시 열람할 수 있는 문서 포털을 제공합니다.

3) 주요 업무
- 다중 법인 통합 모니터링 대시보드 화면: 법인별 데이터 집계를 PostgreSQL Materialized View로 처리하고, Redis 캐시로 집계 지연을 최소화하며, Recharts 기반 멀티 시리즈 차트로 법인 간 비교 분석 화면을 구현합니다.
- 파레토 분석 및 임계치 알림 시스템 화면: 불량 데이터 누적 집계를 주기적 집계 잡으로 처리하고, 임계치 룰을 DB에서 동적으로 관리하며, Bull Queue를 통해 알림을 비동기 발송하여 검사 입력 성능에 영향이 없도록 설계합니다.
- Google Sheets 연동 및 데이터 수집 자동화 화면: Google Sheets API v4를 통해 지정된 시트의 신규 행을 주기적으로 수집하고, 데이터 검증·변환 후 QMS DB에 upsert하며, 동기화 상태와 오류 건을 관리자 화면에서 확인할 수 있도록 구현합니다.
- QR 코드 기반 표준문서 열람 포털 화면: 표준문서를 HTML 템플릿으로 게시하고 문서별 고유 QR 코드를 생성하여, 비로그인 상태에서도 해시 기반 토큰으로 접근 가능한 공개 열람 URL 구조를 구현합니다.

4) 주안점
- 법인별 데이터 파티셔닝 전략을 적용하여 단일 쿼리로 전체 법인 집계가 가능한 DB 구조를 설계하고, 실시간 WebSocket 이벤트로 신규 불량 입력 시 대시보드가 자동 갱신되도록 구현합니다.
- 임계치 평가 로직을 Rule 패턴으로 추상화하여 새로운 불량 유형이나 알림 조건 추가 시 코드 변경 없이 DB 룰 테이블만 갱신하면 적용되는 확장 가능한 알림 엔진을 설계합니다.
- Google Sheets 수집 파이프라인을 Adapter 패턴으로 설계하여 향후 Google Form, CSV 직접 업로드 등 다양한 입력 소스를 동일한 변환·저장 파이프라인으로 처리할 수 있는 확장 구조를 구현합니다.

※ 본 포트폴리오는 해당 도메인의 핵심 기능을 직접 설계·구현한 구현 제안 데모입니다. 기획부터 DB 설계, API, 프론트엔드까지 전 과정을 1인 풀스택으로 구현하며, 결과물 URL에서 실제 동작을 확인하실 수 있습니다. 프로젝트 착수 시 이 데모를 기반으로 빠르게 개발을 진행할 수 있습니다.

### 프로젝트 성과 (achievements)

#### 성과 1
- 성과명: Materialized View 기반 KPI 집계 파이프라인 설계
- 설명: 법인별·라인별·월별 품질 KPI를 PostgreSQL Materialized View 6개로 집계, Redis TTL 캐시와 write-through 전략으로 조회 성능을 확보합니다.

#### 성과 2
- 성과명: Rule 패턴 임계치 알림 엔진 구현
- 설명: 알림 조건을 DB 룰 테이블로 외부화하여 코드 변경 없이 새 불량 유형 알림 추가가 가능한 확장 설계, Bull Queue 다채널 발송 큐 구현합니다.

#### 성과 3
- 성과명: Google Sheets Adapter 수집 파이프라인 설계
- 설명: Google Sheets API v4 연동 + externalId 기반 중복 방지 upsert, Adapter 패턴으로 CSV 등 추가 입력 소스를 동일 파이프라인으로 처리하는 구조를 구현합니다.

#### 성과 4
- 성과명: QR 토큰 기반 문서 접근 제어 구조 구현
- 설명: HMAC-SHA256 기반 영구 토큰 + DB 무효화 테이블 구조로 보안과 접근 편의성을 동시에 확보, PWA 캐싱으로 오프라인 문서 열람 지원을 구현합니다.
